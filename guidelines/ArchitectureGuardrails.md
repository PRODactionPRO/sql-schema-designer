# Architecture Guardrails (Phase 0)

## Цель
Сделать проект предсказуемым и устойчивым без сложной инфраструктуры.

## 1) Минимальная FSD-иерархия
- `app` — запуск приложения, роутинг, глобальные провайдеры.
- `pages` — экраны и их локальная оркестрация.
- `shared` — общие типы, утилиты, UI-примитивы, конфиг.

## 2) Правила импортов
- `shared` не импортирует из `pages` и `app`.
- `pages` не импортирует из `app`.
- Нарушения блокируются ESLint.

## 3) Правила хранения данных
- Прямой доступ к `localStorage` запрещен вне `shared/lib/project-storage.ts`.
- Любая загрузка/импорт JSON проходит через runtime-нормализацию:
  - `normalizeSchema`
  - `normalizeProjectData`

## 4) Guardrails качества
- CI на GitHub: `typecheck` + `lint` + `build`.
- Обязательная проверка перед merge в `main`.

## 5) Технический долг (оставлен осознанно)
- Часть legacy-правил React hooks пока в `warn/off`, чтобы не тормозить доставку.
- Полный вынос editor-store на slice-архитектуру отложен на следующий этап.
